Inline, block, or maybe inline-block?

Each of the HTML elements has a default „display” property. For example, all of the text will always be floating letter by letter, even if we add anchor elements to it. That’s because its default display property is „inline”. Elements displayed inline cannot have height or length changed via CSS.

Let’s look at the example below. The „a” element has its’ default display property set as „inline”. When I change the height of the element to 100px, nothing happens. That’s because I cannot change height of an inline element.

What is going to happen when I change display property to block? The link changes into a separate block and places itself in a separate space that can have its’ height and length changed with css. Since I left the height at 100px, changing display property created a box of this height with the „a” element inside of it.

Now I changed a display to inline-block. This one joins the best of previous properties. So my „a” element is back in line with the text, but also has a heigth property set to 100px.

The example of „a” element may not sound too useful - who wants to create a box for a link? Try to change that property for an image - it will give you a lot more interesting results.
